HTML

  1. DOCTYPE意义
    !DOCTYPE，一个文档类型标记是一种标准通用标记语言的文档类型声明，它的目的是要告诉标准通用标记语言解析器，它应该使用什么样的文档类型定义（DTD）来解析文档。

  2. html和xhtml区别
    HTML是一种基本的WEB网页设计语言，XHTML是一个基于XML的置标语言，看起来与HTML有些相象，只有一些小的但重要的区别，XHTML就是一个扮演着类似HTML的角色的XML，所以，本质上说，XHTML是一个过渡技术，结合了XML(有几分)的强大功能及HTML(大多数)的简单特性。
    HTML 和 XHTML 的区别简单来说，XHTML 可以认为是 XML 版本的 HTML，为符合 XML 要求，XHTML 语法上要求更严谨些。
    以下是 XHTML 相对 HTML 的几大区别：
    XHTML要求正确嵌套
    XHTML所有元素必须关闭
    XHTML区分大小写
    XHTML属性值要用双引号
    XHTML用 id 属性代替 name 属性
    XHTML特殊字符的处理

  3. 块级元素与行内元素的区别有哪些？

    • 在标准文档流里面，块级元素具有以下特点：
      总是在新行上开始，占据一整行；
      高度，行高以及外边距和内边距都可控制；
      宽带始终是与浏览器宽度一样，与内容无关；
      它可以容纳内联元素和其他块元素。
    • 行内元素的特点：
      和其他元素都在一行上；
      行高和水平方向的外边距与内边距部分可改变；
      宽度只与内容有关；
      行内元素只能容纳文本或者其他行内元素。
      不可以设置宽高，其宽度随着内容增加，高度随字体大小而改变，内联元素可以设置margin，但是margin不对上下起作用，只能对左右起作用，也可以设置padding，但是padding在ie6中不对上下起作用，只能对左右起作用

  4. HTML5里的input有哪些类型？

    • email
    • url
    • number
    • range
    • Date pickers (date, month, week, time, datetime, datetime-local)
    • search
    • color

  5. img标签的title和alt分别是什么不同？

    1. 含义不同
      alt是当图片不存在时的替代文字
      title是对图片的描述与进一步说明
    2. 在浏览器中的表现不同
      　　在firefox和ie8中，当鼠标经过图片时title值会显示，而alt的值不会显示;只有在ie6中，当鼠标经过图片时title和alt的值都会显示。
      　　对于网站seo优化来说，title与alt还有最重要的一点：
      　　搜索引擎对图片意思的判断，主要靠alt属性。所以在图片alt属性中以简要文字说明，同时包含关键词，也是页面优化的一部分。条件允许的话，可以在title属性里，进一步对图片说明。

  6. 理解html语义化，有什么优点？
    根据内容的结构化（内容语义化），选择合适的标签（代码语义化）便于开发者阅读和写出更优雅的代码的同时让浏览器的爬虫和机器很好地解析。
    优点：

    1. 去掉或者丢失样式的时候能够让页面呈现出清晰的结构；
    2. 有利于SEO：和搜索引擎建立良好沟通，有助于爬虫抓取更多的有效信息：爬虫依赖于标签来确定上下文和各个关键字的权重；
    3. 方便其他设备解析（如屏幕阅读器、盲人阅读器、移动设备）以意义的方式来渲染网页；
    4. 便于团队开发和维护，语义化更具可读性，是下一步吧网页的重要动向，遵循W3C标准的团队都遵循这个标准，可以减少差异化。

CSS特性

  1. 怎么样让元素变透明，怎么隐藏元素？两者有什么区别？
    变透明：
    visibility:hidden;
    或

    隐藏元素：display:none;
    区别：

    1. opacity=0，该元素隐藏起来了，但不会改变页面布局，并且，如果该元素已经绑定一些事件，如click事件，那么点击该区域，也能触发点击事件的
    2. visibility=hidden，该元素隐藏起来了，但不会改变页面布局，但是不会触发该元素已经绑定的事件
    3. display=none，把元素隐藏起来，并且会改变页面布局，可以理解成在页面中把该元素删除掉一样

  5. inline inline-block什么区别？
    display:inline-block简单来说就是将对象呈现为inline对象，但是对象的内容作为block对象呈现。之后的内联对象会被排列在同一行内。比如我们可以给一个link（a元素）inline-block属性值，使其既具有block的宽度高度特性又具有inline的同行特性。

  7. 什么是响应式设计与媒体查询？
    响应式Web设计(Responsive Web design)的理念是：页面的设计与开发应当根据用户行为以及设备环境(系统平台、屏幕尺寸、屏幕定向等)进行相应的响应和调整。具体的实践方式由多方面组成，包括弹性网格和布局、图片、CSS media query的使用等。无论用户正在使用笔记本还是iPad，我们的页面都应该能够自动切换分辨率、图片尺寸及相关脚本功能等，以适应不同设备；换句话说，页面应该有能力去自动响应用户的设备环境。响应式网页设计就是一个网站能够兼容多个终端——而不是为每个终端做一个特定的版本。这样，我们就可以不必为不断到来的新设备做专门的版本设计和开发了。

    媒体查询有两种基本规则：

    2. 使用媒体查询规则。

      right : 不允许右边有浮动对象
      both : 不允许有浮动对象

  10. css hack的方式？
    属性前缀法(即类内部Hack)：例如 IE6能识别下划线_和星号*，IE7能识别星号*，但不能识别下划线_，IE6~IE10都认识”\9”，但firefox前述三个都不能认识。
    选择器前缀法(即选择器Hack)：例如 IE6能识别*html .class{}，IE7能识别*+html .class{}或者*:first-child+html .class{}。
    IE条件注释法(即HTML条件注释Hack)：针对所有IE(注：IE10+已经不再支持条件注释)： <!--[if IE]>IE浏览器显示的内容 <![endif]-->，针对IE6及以下版本： <!--[if lt IE 6]>只在IE6-显示的内容 <![endif]-->。这类Hack不仅对CSS生效，对写在判断语句里面的所有代码都会生效。

CSS布局总结

Share

在前端开发中，接到设计稿和交互稿后第一个问题就是，为了实现网页的需求如何布局的问题。在第一篇博客中，我就来总结一下前端常用的CSS布局方法。

所有CSS布局技术的根本都是3个基本概念：定位、浮动和外边距操纵。

一、水平居中

  • 父元素不定宽，子元素定宽

    1. margin:auto

      基本上会点CSS的人都知道只需定义容器div的宽度，然后将水平外边距设置为auto就能实现容器div的居中

      这在所有现代浏览器中都是有效的。但是，混杂模式的IE 5.x和IE 6不支持margin:auto，于是我们需要使用一种hack方式：

  • 父子元素均不定宽

    1. inline-block + text-align

    2. table + margin

    3. absolute + transform

    4. flex + justify-content

      或

二、垂直居中

  • 父子元素均不定宽

    1. table-cell + vertical-align

    2. absolute + transform

    3. flex + align-items

三、结合以上，实现水平和垂直均居中

  • 父子元素均不定宽

    1. inline-block + text-align + table-cell + vertical-align

    2. absolute + transform

    3. flex + justify-content + align-items

四、两列布局

  • 左右两列均定宽的两列布局

    1. float
      在创建基于浮动的布局时，一般将两列都向左浮动，然后使用外边距或内边距在两列之间创建一个隔离带。在使用这种方法时，列在可用空间内包得很紧，没有喘息的空间。如果浏览器表现良好的话，这不是问题；但是差劲的浏览器会打乱紧密的布局，迫使一列退到另一列的下面。

      IE上就会发生这种情况，因为IE考虑元素内容的尺寸，而不是元素本身的尺寸。在符合标准的浏览器中，如果元素的内容太大，它只会超出框之外。但是，在IE上，如果元素的内容太大，整个元素就会扩展。这可能是有很小的东西引起的，比如把一些文本设置为斜体。如果这发生在非常紧密的布局中，那么就没有足够的空间可以让元素并排出现，有一个浮动元素就会退到下面去。其他IE bug（比如3像素文本偏移bug和双外边距浮动bug）以及各种浏览器取整错误也会导致浮动元素下降。

      为了防止发生这种情况，需要避免浮动布局在包含它们的元素中太挤。可以不使用水平外边距或内边距来建立隔离带，而是让一个元素向左浮动，让另一个元素向右浮动，从而创建视觉上的隔离、如果一个元素的尺寸意外地增加了几个像素，那么它不会立刻占满水平空间并下降，而是扩展到视觉隔离带中。

  • 一列定宽，另一列自适应的两列布局

    1. float + margin

    2. float + margin + (fix)

    3. float + overflow

    4. table

    5. flex

  • 一列不定宽，另一列自适应的两列布局

    1. float + overflow

    2. table

    3. flex

五、三列布局——多列布局

  • 三列均定宽的三列布局
    与两列均定宽布局方法相似，唯一的差异是在内容div中添加两个新的div。还可以重用灵活的primary和secondary类名。

  • 两列定宽，一列自适应的三列布局
    与一列定宽，一列自适应的两列布局方法相似
    如：float + overflow

  • 两列不定宽，一列自适应的三列布局
    与一列不定宽，一列自适应的两列布局方法相似
    如：float + overflow

六、等分布局

  • 多列等分布局

    1. float

    2. table

    3. flex

七、固定宽度、流式和弹性布局

固定宽度布局以像素为单位。

流式布局以百分数为单位，这种布局能够相对于浏览器窗口进行伸缩。工具Liquid Fold 可以查看浏览器统计数据确定最常用的窗口尺寸，然后根据固定宽度版本与这个尺寸的比例，选择容器百分数。

弹性布局相对于字号来设置元素的宽度，以em为单位设置宽度，可以确保在字号增加时整个布局随之扩大。另外，因为在文本字号增加时整个布局会加大，所以弹性布局会变得比浏览器窗口宽，导致水平滚动条的出现。为了防止这种情况，可能需要在容器div上添加100%的max-width。IE6和更低版本不支持max-width，但是较新的版本都支持它。如果要在IE6中支持max-width，可以使用JavaScript。

小技巧应用弹性布局：大多数浏览器上的默认字号是16像素，10像素大约是16像素的62.5%，所以在主体上将font-size设置为62.5%就可以了。

最佳实践：内部宽度使用百分数，只以em为单位设置容器的宽度。

八、fuax列

在布局的最大高度的元素（比如一个容器div）上应用重复的背景图像。为固定宽带的设计创建faux列非常容易，因为我们知道列的尺寸和位置。为流式布局创建faux列的技巧是按百分比对背景图像进行定位。

九、等高布局

  1. table-天然等高

  2. flex-天然等高

  3. float + overflow伪等高

    这种技术的关键是给每个框设置大的底内边距，然后用数值相似的负外边距消除这个高度，最后设置父容器overflow:hidden，列就在最高点被裁切。
    如果需要给底部背景设置圆角效果，则需要运用绝对定位应用背景图像。

十、CSS3 列

CSS3也可以创建等高文本列，这要通过column-count、column-width、column-gap属性实现。
其优点是在可用空间小于已定义列的宽度时的处理方式。列不会像使用浮动时那样回绕，而是会减少列数。因此，如果空间不够显示三列，就会减少到两列。缺点是兼容性差，而且column-rule存在不跟随页面放缩而移动的bug。

十一、全屏布局

  • position(不兼容IE6，可用Hack方案 使之兼容)

  • flex(不兼容IE9及以下)

  • 可以将上述两种方法以px为单位变成以百分比为单位，实现流式全局布局。

十一、全自适应的全屏布局

  • position不能实现全自适应

  • flex只需将定高宽的地方去掉即可，根据内容来自适应。

  • grid草案
