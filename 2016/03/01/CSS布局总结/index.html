CSS布局总结

2016-03-01
  • CSS

在前端开发中，接到设计稿和交互稿后第一个问题就是，为了实现网页的需求如何布局的问题。在第一篇博客中，我就来总结一下前端常用的CSS布局方法。

所有CSS布局技术的根本都是3个基本概念：定位、浮动和外边距操纵。

一、水平居中

  • 父元素不定宽，子元素定宽

    1. margin:auto

      基本上会点CSS的人都知道只需定义容器div的宽度，然后将水平外边距设置为auto就能实现容器div的居中

      这在所有现代浏览器中都是有效的。但是，混杂模式的IE 5.x和IE 6不支持margin:auto，于是我们需要使用一种hack方式：

  • 父子元素均不定宽

    1. inline-block + text-align

    2. table + margin

    3. absolute + transform

    4. flex + justify-content

      或

二、垂直居中

  • 父子元素均不定宽

    1. table-cell + vertical-align

    2. absolute + transform

    3. flex + align-items

三、结合以上，实现水平和垂直均居中

  • 父子元素均不定宽

    1. inline-block + text-align + table-cell + vertical-align

    2. absolute + transform

    3. flex + justify-content + align-items

四、两列布局

  • 左右两列均定宽的两列布局

    1. float
      在创建基于浮动的布局时，一般将两列都向左浮动，然后使用外边距或内边距在两列之间创建一个隔离带。在使用这种方法时，列在可用空间内包得很紧，没有喘息的空间。如果浏览器表现良好的话，这不是问题；但是差劲的浏览器会打乱紧密的布局，迫使一列退到另一列的下面。

      IE上就会发生这种情况，因为IE考虑元素内容的尺寸，而不是元素本身的尺寸。在符合标准的浏览器中，如果元素的内容太大，它只会超出框之外。但是，在IE上，如果元素的内容太大，整个元素就会扩展。这可能是有很小的东西引起的，比如把一些文本设置为斜体。如果这发生在非常紧密的布局中，那么就没有足够的空间可以让元素并排出现，有一个浮动元素就会退到下面去。其他IE bug（比如3像素文本偏移bug和双外边距浮动bug）以及各种浏览器取整错误也会导致浮动元素下降。

      为了防止发生这种情况，需要避免浮动布局在包含它们的元素中太挤。可以不使用水平外边距或内边距来建立隔离带，而是让一个元素向左浮动，让另一个元素向右浮动，从而创建视觉上的隔离、如果一个元素的尺寸意外地增加了几个像素，那么它不会立刻占满水平空间并下降，而是扩展到视觉隔离带中。

  • 一列定宽，另一列自适应的两列布局

    1. float + margin

    2. float + margin + (fix)

    3. float + overflow

    4. table

    5. flex

  • 一列不定宽，另一列自适应的两列布局

    1. float + overflow

    2. table

    3. flex

五、三列布局——多列布局

  • 三列均定宽的三列布局
    与两列均定宽布局方法相似，唯一的差异是在内容div中添加两个新的div。还可以重用灵活的primary和secondary类名。

  • 两列定宽，一列自适应的三列布局
    与一列定宽，一列自适应的两列布局方法相似
    如：float + overflow

  • 两列不定宽，一列自适应的三列布局
    与一列不定宽，一列自适应的两列布局方法相似
    如：float + overflow

六、等分布局

  • 多列等分布局

    1. float

    2. table

    3. flex

七、固定宽度、流式和弹性布局

固定宽度布局以像素为单位。

流式布局以百分数为单位，这种布局能够相对于浏览器窗口进行伸缩。工具Liquid Fold 可以查看浏览器统计数据确定最常用的窗口尺寸，然后根据固定宽度版本与这个尺寸的比例，选择容器百分数。

弹性布局相对于字号来设置元素的宽度，以em为单位设置宽度，可以确保在字号增加时整个布局随之扩大。另外，因为在文本字号增加时整个布局会加大，所以弹性布局会变得比浏览器窗口宽，导致水平滚动条的出现。为了防止这种情况，可能需要在容器div上添加100%的max-width。IE6和更低版本不支持max-width，但是较新的版本都支持它。如果要在IE6中支持max-width，可以使用JavaScript。

小技巧应用弹性布局：大多数浏览器上的默认字号是16像素，10像素大约是16像素的62.5%，所以在主体上将font-size设置为62.5%就可以了。

最佳实践：内部宽度使用百分数，只以em为单位设置容器的宽度。

八、fuax列

在布局的最大高度的元素（比如一个容器div）上应用重复的背景图像。为固定宽带的设计创建faux列非常容易，因为我们知道列的尺寸和位置。为流式布局创建faux列的技巧是按百分比对背景图像进行定位。

九、等高布局

  1. table-天然等高

  2. flex-天然等高

  3. float + overflow伪等高

    这种技术的关键是给每个框设置大的底内边距，然后用数值相似的负外边距消除这个高度，最后设置父容器overflow:hidden，列就在最高点被裁切。
    如果需要给底部背景设置圆角效果，则需要运用绝对定位应用背景图像。

十、CSS3 列

CSS3也可以创建等高文本列，这要通过column-count、column-width、column-gap属性实现。
其优点是在可用空间小于已定义列的宽度时的处理方式。列不会像使用浮动时那样回绕，而是会减少列数。因此，如果空间不够显示三列，就会减少到两列。缺点是兼容性差，而且column-rule存在不跟随页面放缩而移动的bug。

十一、全屏布局

  • position(不兼容IE6，可用Hack方案 使之兼容)

  • flex(不兼容IE9及以下)

  • 可以将上述两种方法以px为单位变成以百分比为单位，实现流式全局布局。

十一、全自适应的全屏布局

  • position不能实现全自适应

  • flex只需将定高宽的地方去掉即可，根据内容来自适应。

  • grid草案
